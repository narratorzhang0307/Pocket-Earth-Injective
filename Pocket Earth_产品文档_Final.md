# Pocket Earth · 口袋地球 产品文档

## 0. 文档信息

产品名称：Pocket Earth · 口袋地球

版本：Demo v1 / PWA 已上线

形态：移动端优先 Web App / PWA，云端自托管服务 + 端侧能力可插拔

线上地址：https://pocketearth.throughtheglass.art

目标：把书、电影、音乐、照片、行程、心情等个人碎片重新钉回它们在地球上的坐标，用一颗会长大的个人地球，构建本地空间知识库与端云双脑 agent 框架。

## 1. 背景与问题

用户的个人记忆长期散落在十几个 App 中：书在豆瓣，电影在片单，照片在相册，行程在备忘录，心情在聊天框或日记里。记录越多，越难被重新唤起。

核心问题包括：

- 缺少统一索引：时间、标题、平台都无法把所有碎片真正串起来。
- 记忆检索弱：人往往记得“在哪”，却忘记“哪一天”或“叫什么”。
- 记录沉底：碎片被存下之后不再被使用，越积越懒得整理。
- 推荐不懂人：工具只统计数量，很难识别“读得少但极爱”的真实口味。
- 隐私门槛高：照片、票据、定位、证件信息不适合整张上传给云端。

Pocket Earth 选择用地理坐标作为记忆主键，把“地球”变成个人知识库的索引层。

## 2. 产品定位

Pocket Earth 是一款“本地空间知识库 x 多智能体编排 x 端云双脑”的个人记忆产品。

它不是单纯地图，也不是普通聊天框，而是一颗持续生长的私人地球：每次记录一本书、一部电影、一首歌、一张照片、一段行程或一句心情，地球上都会多出一个可回看的点。

产品的精神内核来自罗杰·泽拉兹尼《趁生命气息逗留》中的 Frost：一台想理解人类的机器，通过收集碎片理解“成为一个人”的感觉。Pocket Earth 中的 frost-agent 继承这种气质：克制地委派、耐心地沉淀，以理解用户为目的，而不是占有或替用户做主。

## 3. 核心概念

### 3.1 地理空间锚定

六类个人碎片被统一锚定为同一种空间对象：

- 书：故事地 / 作者地
- 电影：取景地 / 国别
- 音乐：歌手出身城 / 歌中城市
- 照片：EXIF 经纬度或照片语义地点
- 行程：私人足迹和路线
- 心情：一句话判断地名并落点

地理坐标是锚，不是背景图。缩放或平移时，标记依然固定在真实坐标上。

### 3.2 三个入口

- Photos：时间、日历、杂志三视图相册。杂志视图像双页手帐，灰度触碰变彩色。
- My Map：矢量地球，承载音乐、照片、电影、书、行程、心情和用户星球。
- Agents：frost-agent 多智能体控制台，包含六大 agent、圆桌议事、造物主和万能记一笔。

### 3.3 frost-agent 内核

总 frost-agent 扮演 CEO，不亲自吞下所有原始数据，而是把不同对象委派给专门的子 agent。每个 agent 在独立上下文中完成“整理、定位、产出落点”，再把结构化结论回流。

关键原则：

- 隔离：整理相册、抓网页、解析票据等噪声停留在子 agent 内。
- 约束：每个子 agent 只拿必要工具，动作必须经过 Boundary 校验。
- 可组合：agent 契约化、可并行委派，新增能力只需注册一个 agent 或 skill。
- 可观测：运行过程通过 trace 展示 router、agent、skill 每一跳。

## 4. 核心体验

### 4.1 记一笔，地球立刻多一个点

用户在任意 agent 中记录电影、书、行程、星球或心情后，产出会写入统一 store，地球订阅后实时重绘，无需刷新页面。

### 4.2 万能 Jot Agent

用户不需要先判断该找哪个 agent。把一句话或截图丢进 Jot，frost 会判断它是书、影、行程还是心情，并自动委派给对应 agent。确认后才落点，保证自动化不越权。

### 4.3 长期记忆闭环

产品把会话记忆和跨会话长期画像拆开管理：memory 保持当下连贯，profile 沉淀长期偏好。每次 agent 运行结束后，会把新偏好补全字段写回画像，下次推荐自动避开已看内容，并优先识别真正偏爱的地区、作者、导演和流派。

### 4.4 多 agent 圆桌

用户可以组织读书官、影评人、选曲师、摄影眼、旅人、造星者、抬杠侠等 agent 进入圆桌，以自由辩论、法庭、头脑风暴等模式讨论问题。Frost 作为庭长负责收敛与裁断。

## 5. 功能需求

### 5.1 地球索引

- 支持音乐、照片、电影、书、行程、心情与用户星球多图层。
- 标记颜色和形状区分类型，用户星球以圆点与方块标记区分。
- 放大到街道级后展示拍立得照片贴、心情贴和详情卡。
- 标记随经纬度锚定，不随屏幕滚动漂移。

### 5.2 相册与端侧整理

- 根据时间戳自动派生时间、日历、杂志三视图。
- 列表使用缩略图快速加载，详情展示高清图。
- 端侧模型对照片做收藏价值评分，原图不出端。
- 高价值照片按经纬度钉到地球，重复照片只标记建议，不直接删除。

### 5.3 Agent 与 Skill

- music-agent：把音乐钉到歌手出身地或歌曲城市。
- books-agent：把书钉到故事地或作者地。
- movies-agent：把电影钉到取景地或国别。
- photos-agent：整理相册并钉选中照片。
- travel-agent：规划或归档行程。
- jot-agent：统一入口，自动识别书、影、行程和心情。
- AgentForge：以声明式方式生成新 agent，但只能路由到白名单能力。

### 5.4 端云双脑

- 端侧 Selector 管“挑和找”：意图预分类、选歌选图排序、嵌入检索、截图打标。
- 云端 Brain 管“写”：自由文本生成、推荐理由、长文回答。
- 无 LLM、无端侧、无网络时都提供规则兜底，确保产品不白屏。

## 6. 技术方案

前端：React + TypeScript + Vite + Tailwind v4 + Mapbox GL / 地球可视化，移动端 430 x 932 比例优先，可安装为 PWA。

后端：自托管 Node 服务，Nginx 反代，静态托管 dist，并提供云脑、端侧、抓图等 API。

模型与编排：provider-compat 统一云脑请求，EdgeModel 契约封装端侧能力。端侧可接 WebLLM、Ollama、MNN x Qwen，云端使用 Qwen / DeepSeek 等服务端密钥模型。

数据：已接入 2124 部观影、1055 本阅读、619 位艺人、536 张照片。数据源与实现解耦，替换数据即可自动重排三视图、地图点和分类。

工程韧性：注册表化 router、fingerprint 缓存、指数退避重试、结构化输出 schema 约束、SSE 逐 token 流式输出、health trace 追踪。

## 7. 风险与边界

- 隐私：照片、票据、证件和定位信息优先端侧处理，敏感数据不整图上云。
- 模型幻觉：所有落点动作必须 suggest-then-confirm，并经过 Boundary 校验。
- 成本与延迟：端侧承担筛选和检索，云端只承担需要语言生成的任务。
- 复杂度：多 agent 容易膨胀，必须通过 skill 契约、白名单和 trace 保持可维护。

## 8. 后续迭代

- 真 WebSocket 跨端同步，让手机、桌面和未来硬件共享运行状态。
- RAG 知识库增强，让书影乐内容回答更准确。
- 主动 heartbeat 建议引擎，以克制方式提醒用户可记录、可回望的碎片。
- 运行时语音，让 Frost 从文字 agent 进一步变成可对话的空间记忆伙伴。
